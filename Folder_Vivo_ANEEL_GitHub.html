● Monitor ativo · curadoria automática

A ANEEL é o regulador. Mas o volume é tanto que vira ruído. Este monitor mostra só o que tem score SOA ≥ 5 — atos com prazo, com vetor afetado, com cliente concreto. O resto fica no banco interno.

Cada cartão renderiza o `score_soa` (cinza) ao lado do status (cor). Banco interno cobre todas REN, REH, CP, despachos e PRODIST/Procedimentos de Rede. Só sobe aqui o que dispara ação comercial em vetores ativos da SOA. Quando um ato cai abaixo do threshold, vai para `_historico_caidos` (não-exibido).

Curadoria, não amontoado. Schema do JSON estendido com `score_soa` e `vetores_afetados` por fato (Arquitetura_Monitores_Curados_SOA_v1.md). Threshold canônico: score ≥ 5. Cada varredura reavalia: o que sobe, o que cai, o que entra/sai do destaque.
