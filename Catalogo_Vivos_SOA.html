● Catálogo honesto · vivo × estático

Nem todo material é "vivo". Aqui eles estão separados pelo que realmente são: os que se atualizam sozinhos e os que são editados à mão.

Vivos — folders que fazem fetch de um JSON canônico a cada visita; quando o monitor regulatório atualiza o JSON, o folder reflete na hora. Têm selo de frescor (verde/amarelo/vermelho). Referência — materiais estáticos, atualizados manualmente; o carimbo mostra a data da última edição real — sem alarme de "defasado", porque material estático não fica defasado, fica estável.

Como o frescor é medido. Cada cartão consulta a API pública do GitHub (`api.github.com/repos/.../commits?path=…`) ao abrir, e mostra o timestamp do último commit que tocou aquele arquivo. Nos vivos, esse timestamp é a última regeneração pelo monitor. Nos de referência, é a data da última edição manual. O carimbo no canto é a hora em que você abriu este catálogo.
